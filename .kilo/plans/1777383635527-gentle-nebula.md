- **Image**: `ghcr.io/open-webui/open-webui:main-slim`
- **Ports**: `9707:8080` (host:container)

  - `./data:/app/backend/data` — local DATA_DIR mount (persistent data, uploads, vector DB)

### Environment Variables (defaults added in .env.default)
```yaml
services:
  openwebui:
    # ... image, ports, volumes, networks ...
    env_file:
      - .env
    environment:
      # --- Security ---
      - WEBUI_SECRET_KEY=${WEBUI_SECRET_KEY}

      # --- OpenAI-Compatible Provider (nano-gpt.com) ---
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - OPENAI_API_BASE_URL=${OPENAI_API_BASE_URL:-https://nano-gpt.com/api/v1}

   - ENABLE_OLLAMA_API=${ENABLE_OLLAMA_API:-false}

      # --- Multi-User Settings ---
      - ENABLE_SIGNUP=${ENABLE_SIGNUP:-False}
      - ENABLE_PASSWORD_AUTH=${ENABLE_PASSWORD_AUTH:-True}

      # --- API Keys (for programmatic access, e.g. workflows) ---
      - ENABLE_API_KEYS=${ENABLE_API_KEYS:-True}

      # --- Tool Calling (Essentials) ---
      - ENABLE_TOOL_CALLING=${ENABLE_TOOL_CALLING:-True}
      - DEFAULT_FUNCTION_CALLING_MODE=${DEFAULT_FUNCTION_CALLING_MODE:-native}

      # --- Default Model (Main Chat) ---
      - DEFAULT_MODELS=${DEFAULT_MODELS:-openai:moonshotai/kimi-k2.6}

      # --- Background Task Model ---
      # Uses TASK_MODEL_EXTERNAL (not TASK_MODEL) because we use an external OpenAI-compatible provider, not local Ollama
      - TASK_MODEL_EXTERNAL=${TASK_MODEL_EXTERNAL:-openai:unsloth/gemma-3-4b-it}
      - ENABLE_AUTOCOMPLETE_GENERATION=${ENABLE_AUTOCOMPLETE_GENERATION:-True}
      - ENABLE_FOLLOW_UP_GENERATION=${ENABLE_FOLLOW_UP_GENERATION:-True}
      - ENABLE_TITLE_GENERATION=${ENABLE_TITLE_GENERATION:-True}

      # --- Image Generation (Wavespeed via OpenAI-compatible engine) ---
      - ENABLE_IMAGE_GENERATION=${ENABLE_IMAGE_GENERATION:-True}
      - IMAGE_GENERATION_ENGINE=${IMAGE_GENERATION_ENGINE:-openai}
      - IMAGE_GENERATION_MODEL=${IMAGE_GENERATION_MODEL:-nano-banana-2}
      # OpenWebUI OpenAPI-compatible image gen uses IMAGES_OPENAI_* variables
      - IMAGES_OPENAI_API_BASE_URL=${IMAGES_OPENAI_API_BASE_URL:-https://api.wavespeed.ai/v1}
      - IMAGES_OPENAI_API_KEY=${IMAGES_OPENAI_API_KEY:-${WAVESPEED_API_KEY:-${OPENAI_API_KEY}}}

      # --- RAG / Knowledge Base ---
      # Uses nano-gpt.com's OpenAI-compatible embeddings (external, no local processing)
      - RAG_EMBEDDING_ENGINE=${RAG_EMBEDDING_ENGINE:-openai}
      - RAG_OPENAI_API_BASE_URL=${RAG_OPENAI_API_BASE_URL:-${OPENAI_API_BASE_URL}}
      - RAG_OPENAI_API_KEY=${RAG_OPENAI_API_KEY:-${OPENAI_API_KEY}}
      - RAG_EMBEDDING_MODEL=${RAG_EMBEDDING_MODEL:-text-embedding-3-small}

      # --- Memory System ---
      - ENABLE_MEMORIES=${ENABLE_MEMORIES:-True}

      # --- Redis / Valkey (from datastore-memory) ---
      # Valkey is Redis-compatible; connects to the existing datastore-memory service on home-lab-net
      - REDIS_URL=${REDIS_URL:-redis://datastore-memory:6379/0}

      # --- Model Cache ---
      - ENABLE_BASE_MODELS_CACHE=${ENABLE_BASE_MODELS_CACHE:-True}
      - MODELS_CACHE_TTL=${MODELS_CACHE_TTL:-300}

      # --- WebUI Customization ---
      - WEBUI_NAME=${WEBUI_NAME:-Luna}
      - WEBUI_URL=${WEBUI_URL:-https://luna.foxbites.net}

# ============================================
# OpenWebUI Configuration (luna stack)
# ============================================

# --- Data Directory (mapped to ./data on the local machine) ---
# Container data (uploads, vector db, cache) will live in ./data
# If you want data stored elsewhere, change this and ensure the volume
# mount in docker-compose.yml matches.
DATA_DIR=/app/backend/data

# --- Security ---
# Generate with: openssl rand -hex 32
WEBUI_SECRET_KEY=CHANGE_ME_generated_secret_key

# --- OpenAI-Compatible Provider (nano-gpt.com for chat LLM) ---
OPENAI_API_KEY=YOUR_NANO_GPT_API_KEY
# Standard OpenAI-compatible base URL; adjust if nano-gpt uses a different path
OPENAI_API_BASE_URL=https://nano-gpt.com/api/v1

# --- Disable local Ollama (we only use external providers) ---
ENABLE_OLLAMA_API=false

# --- Admin Account (created on first startup) ---
# When both email and password are set, admin is created automatically
# and ENABLE_SIGNUP is automatically disabled for security.
WEBUI_ADMIN_EMAIL=admin@yourdomain.local
WEBUI_ADMIN_PASSWORD=CHANGE_ME_strong_admin_password
WEBUI_ADMIN_NAME=Admin

# --- Multi-User Settings ---
ENABLE_SIGNUP=False                    # Disable public registration (admin adds users)
ENABLE_PASSWORD_AUTH=True              # Allow password login (set to False if only using OAuth)

# --- API Keys (for programmatic access, workflows, etc.) ---
ENABLE_API_KEYS=True

# --- Tool Calling (Essentials) ---
ENABLE_TOOL_CALLING=True
DEFAULT_FUNCTION_CALLING_MODE=native   # Native mode recommended for modern models

# --- Default Model (Main Chat) ---
# Kimi K2.6 with thinking enabled
DEFAULT_MODELS=openai:moonshotai/kimi-k2.6:thinking

# --- Background Task Model ---
# Uses TASK_MODEL_EXTERNAL (not TASK_MODEL) because we use an external OpenAI-compatible provider, not local Ollama.
TASK_MODEL_EXTERNAL=openai:unsloth/gemma-3-4b-it
ENABLE_AUTOCOMPLETE_GENERATION=True
ENABLE_FOLLOW_UP_GENERATION=True
ENABLE_TITLE_GENERATION=True

# --- Image Generation (Wavespeed via OpenAI-compatible engine) ---
# Note: Image generation uses wavespeed.ai while chat LLM uses nano-gpt.com
ENABLE_IMAGE_GENERATION=True
IMAGE_GENERATION_ENGINE=openai
IMAGE_GENERATION_MODEL=nano-banana-2   # Wavespeed's nano banana 2 model
# OpenWebUI uses IMAGES_OPENAI_* for OpenAI-compatible image generation
IMAGES_OPENAI_API_BASE_URL=https://api.wavespeed.ai/v1
IMAGES_OPENAI_API_KEY=${WAVESPEED_API_KEY}  # Set WAVESPEED_API_KEY below, or use OPENAI_API_KEY as fallback

# --- Wavespeed API (for image generation) ---
# Get separate API key from https://wavespeed.ai/
WAVESPEED_API_KEY=YOUR_WAVESPEED_API_KEY

# --- RAG / Knowledge Base ---
# Uses nano-gpt.com's OpenAI-compatible embeddings endpoint (external, no local processing)
RAG_EMBEDDING_ENGINE=openai
RAG_OPENAI_API_BASE_URL=${OPENAI_API_BASE_URL}
RAG_OPENAI_API_KEY=${OPENAI_API_KEY}
# Choose embedding model supported by nano-gpt.com (e.g., text-embedding-3-small, bge-m3)
RAG_EMBEDDING_MODEL=text-embedding-3-small

# --- Memory System ---
ENABLE_MEMORIES=True

# --- Redis / Valkey (from datastore-memory stack) ---
# datastore-memory is already running on home-lab-net; OpenWebUI can use it for
# caching, session storage, and background task backends.
REDIS_URL=redis://datastore-memory:6379/0

# --- Model Cache ---
ENABLE_BASE_MODELS_CACHE=True
MODELS_CACHE_TTL=300

# --- WebUI Customization ---
WEBUI_NAME=Luna
# CRITICAL: Set this BEFORE enabling OAuth/SSO; used for redirect URIs
WEBUI_URL=https://luna.foxbites.net
DEFAULT_LOCALE=en
```

## `README.md` Content
```markdown
# Luna (OpenWebUI)

OpenWebUI stack running on port 9707, connected to nano-gpt.com as the AI provider.

## Prerequisites

1. **Docker & Docker Compose** installed
2. **home-lab-net** network created:
   ```bash
   make create-network
   ```
   (from the repository root)

3. **nano-gpt.com API key** and chosen model(s) configured

## Configuration

1. Copy `.env.default` to `.env`:
   ```bash
   cp .env.default .env
   ```

2. Edit `.env` and set:
   - `WEBUI_SECRET_KEY` - generate: `openssl rand -hex 32`
   - `OPENAI_API_KEY` - your nano-gpt.com API key
   - `WAVESPEED_API_KEY` - your wavespeed.ai API key (for image generation)
   - `WEBUI_ADMIN_EMAIL` - admin email
   - `WEBUI_ADMIN_PASSWORD` - strong admin password
   - `WEBUI_URL` - your public URL (`https://luna.foxbites.net`) - **CRITICAL: must be set before enabling OAuth/SSO**
   - Optional: `TASK_MODEL_EXTERNAL` - fast model for background tasks (default: `openai:unsloth/gemma-3-4b-it`)
   - Optional: `RAG_EMBEDDING_MODEL` - embedding model (default: `text-embedding-3-small`)
   - Optional: `DATA_DIR` - data directory inside container (default: `/app/backend/data`); local `./data` is already mounted

3. (Optional) Adjust other settings:
   - `DEFAULT_MODELS` - main chat model (default: `openai:moonshotai/kimi-k2.6:thinking`)
   - `IMAGE_GENERATION_MODEL` - image model (default: `nano-banana-2`)
   - `IMAGES_OPENAI_API_BASE_URL` - wavespeed.ai base URL (default: `https://api.wavespeed.ai/v1`)
   - `ENABLE_OLLAMA_API` - set to `false` (no local Ollama)
   - `REDIS_URL` - datastore-memory Valkey URL (default: `redis://datastore-memory:6379/0`)
   - `ENABLE_BASE_MODELS_CACHE` - cache model list (default: `True`)
   - `MODELS_CACHE_TTL` - cache TTL in seconds (default: `300`)
   - `ENABLE_SIGNUP` - set to `True` to allow user registration

**Important:** Verify that the model IDs (`moonshotai/kimi-k2.6:thinking`, `unsloth/gemma-3-4b-it`, `text-embedding-3-small`) are actually available on your chosen providers. Some providers may use different naming. If a model doesn't appear in the UI, add it manually in **Admin → Connections → OpenAI → Model IDs (Filter)**.

## Access

## Post-Installation: Tool Integrations

OpenWebUI can use your existing search-stack services as **Tools**:

### 1. MCP Server Connections (via Admin UI)

Navigate to **Admin Panel → Settings → Tools → MCP** and add:

#### SearXNG (Web Search)
- Name: `SearXNG`
- Transport: `SSE`
- URL: `http://searxng:8080/mcp/sse` (if SearXNG has MCP enabled)
- Note: SearXNG may need MCP plugin enabled

#### GPT Researcher (Research Agent)
- Name: `GPT Researcher`
- Transport: `SSE`
- URL: `http://gpt-researcher:8000/mcp/sse`

#### Crawl4AI (Web Crawler)
- Name: `Crawl4AI`
- Transport: `SSE`
- URL: `http://crawl4ai:11235/mcp/sse`

### 2. OpenAPI Tool Connections (Alternative)

If MCP is not available, connect via **Admin Panel → Connections → OpenAPI**:

- **SearXNG API**: `http://searxng:8080` (Swagger/OpenAPI spec)
- **GPT Researcher API**: `http://gpt-researcher:8000` (OpenAPI spec)
- **Crawl4AI API**: `http://crawl4ai:11235` (OpenAPI spec)

### 3. Enable Tools

After adding connections:
- Go to **Admin Panel → Settings → Tools**
- Enable each tool for use in chats
- Tools appear in the `+` menu in chat input

## Optional: FusionAuth OAuth SSO (via Admin UI)

If you have FusionAuth running in your homelab, OpenWebUI supports OIDC SSO natively.

### Prerequisites
1. Cloudflare tunnel active with `WEBUI_URL=https://luna.foxbites.net` set (required for redirect URI)
2. FusionAuth application configured with:
   - **Client ID** & **Client Secret** (create in FusionAuth Application settings)
   - **Redirect URI**: `https://luna.foxbites.net/oauth/oidc/callback`
   - **Logout URL**: `https://luna.foxbites.net`

### Configuration (Admin Panel)
1. Go to **Admin Panel → Settings → OAuth**
2. Enable OAuth: tick **Enable OAuth Sign-up**
3. Fill in:
   - **Client ID**: from FusionAuth
   - **Client Secret**: from FusionAuth
   - **OIDC Discovery URL**: `https://<your-fusionauth>/.well-known/openid-configuration`
   - **Provider Name**: `FusionAuth` (or your preferred label)
4. Save

### Post-OAuth
- Re-authenticate; you should see "Login with FusionAuth" button
- New users can sign up via OAuth; password auth remains available unless `ENABLE_PASSWORD_AUTH=False`
- Admin: optionally configure group/role mapping via `OAUTH_GROUP_CLAIM`, `OAUTH_ADMIN_ROLES`
- See OpenWebUI docs: [SSO & OAuth](https://docs.openwebui.com/troubleshooting/sso/)

## Cloudflare Tunnel (luna.foxbites.net)

When you configure Cloudflare tunnel separately:
1. Point the tunnel to `http://localhost:9707`
2. Set `WEBUI_URL=https://luna.foxbites.net` in `.env` (required for OAuth/SSO if used later)
3. Restart the stack

## Troubleshooting

### Model not showing up
- Verify `OPENAI_API_BASE_URL` and `OPENAI_API_KEY` are correct
- Check connectivity: `curl $OPENAI_API_BASE_URL/models`
- Some providers don't implement `/models` endpoint — add model ID manually in **Admin → Connections → OpenAI**

### MCP connection failures
- Ensure search-stack services are running and on same `home-lab-net` network
- Check MCP endpoints are exposed: `http://gpt-researcher:8000/mcp/sse`
- Some tools require additional configuration in their own `.env` files

### Port conflict
Port 9707 already in use? Change `ports:` mapping in `docker-compose.yml`.

docker volume inspect luna-luna-data

Also back up the `.env` file (contains API keys and admin credentials).

## Design Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| **Port 9707** | Consistent with existing search-stack ports (9704-9706); avoids conflicts |
| **home-lab-net external network** | Reuses existing network; allows communication with search-stack services |
| **Native function calling mode** | Recommended by OpenWebUI essentials; required for built-in system tools |
| **Task model set externally** | Prevents expensive models from wasting cycles on trivial UI tasks |
| **Multi-user with admin pre-created** | Secure by default; no open registration |
| **MCP tool integration post-install** | Tool endpoints may require additional setup; Admin UI is sufficient |
| **RAG via OpenAI-compatible embeddings** | Uses same nano-gpt.com provider; no extra infrastructure |
| **Volume name: `luna-data`** | Namespaced to stack; consistent with repo pattern |
| **Folder name: `luna`** | User requested; keeps naming consistent with `actual`, `beszel`, etc. |
